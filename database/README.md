# OneEmergence — Database Schema

> Supabase Postgres schema for the OneEmergence consciousness portal.

## Overview

This folder contains the complete database schema as ordered SQL scripts. Run them sequentially against a Supabase project (SQL Editor or CLI) to create the full schema.

**Target:** Supabase Postgres (Frankfurt / eu-central-1)
**Auth:** Supabase Auth manages `auth.users` — app tables reference `auth.users(id)` via UUID foreign keys
**ORM:** Drizzle ORM connects to Supabase Postgres for all app queries
**RLS:** Row-Level Security enabled on all user-owned tables from day one

## How to Apply

### Option A: Supabase Dashboard
1. Open your Supabase project → SQL Editor
2. Paste and run each script in order (00 → 10)
3. Skip `10_seed_dev.sql` for production

### Option B: Supabase CLI
```bash
# Concatenate and run all scripts
cat database/0[0-9]_*.sql | supabase db reset --db-url "$DATABASE_URL"

# Or run individually
for f in database/0[0-9]_*.sql; do
  psql "$DATABASE_URL" -f "$f"
done
```

### Option C: Drizzle Kit (after schema sync)
Once the Drizzle schema in `src/lib/db/schema.ts` is updated to match these tables:
```bash
npx drizzle-kit push
```

## Conventions

- **Primary keys:** UUID, generated by `gen_random_uuid()`
- **Timestamps:** `timestamptz`, with `created_at` defaulting to `now()` and `updated_at` managed by trigger
- **User ownership:** All user-owned tables have a `user_id uuid references auth.users(id) on delete cascade`
- **Soft deletes:** Not used. Hard deletes with cascade. GDPR deletion = delete user → cascades everything.
- **Naming:** `snake_case` for all SQL identifiers. Tables are plural nouns.
- **Enums:** Postgres enums for closed sets (mood tags, guide roles, node types, etc.)

## Schema Relationship Map

```
auth.users (Supabase-managed)
  ├── profiles (1:1)
  ├── user_preferences (1:1)
  ├── journal_entries (1:many)
  │     └── [themes extracted → map_nodes]
  ├── guide_conversations (1:many)
  │     ├── guide_messages (1:many)
  │     └── saved_prompt_cards (many, via source_conversation_id)
  ├── map_nodes (1:many)
  │     └── map_edges (many:many via source/target)
  ├── practice_sessions (1:many)
  └── saved_practices (1:many)
```

## MVP vs Future

- Scripts `00`–`05`, `07`–`09`: **MVP (v1/v2)** — needed for portal, journal, guide, map, practice
- Script `06`: **Future (v3)** — collective field tables, clearly marked, safe to skip initially
- Script `10`: **Dev only** — seed data for local development
